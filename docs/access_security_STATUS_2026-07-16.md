# 校园网访问安全优化状态（2026-07-16）

## 目标

保留学校其他电脑查看研究工作台的能力，同时避免校园网内未授权设备触发更新、保存、重新分类、LLM 调用或情景写入等状态变更。

## 本次改动

- 服务仍默认监听 `0.0.0.0:5001`，局域网其他电脑可以读取页面和 API。
- 所有 POST 写操作默认只允许回环地址（`127.0.0.1`、`::1`）执行。
- 远程管理可选择配置 `ADMIN_TOKEN`，请求头使用 `X-Research-Admin-Token`；令牌采用常量时间比较。
- `ALLOW_REMOTE_WRITES=1` 可显式恢复远程写入，但校园网环境不推荐。
- 默认不再返回通配符 CORS；只有设置 `CORS_ORIGIN` 后才允许指定来源跨站访问。
- POST 请求体限制为 1 MiB，降低异常或恶意大请求占用内存的风险。
- JSON 和静态响应增加 `nosniff` 与同源 referrer 策略。
- 浏览器默认的 `/favicon.ico` 请求返回 204，避免各页面产生无意义的 404 控制台错误。
- `/api/health` 返回当前客户端的 `manage` 或 `read_only` 模式；共享导航和首页显示访问模式，首页显示实际访问地址，不再固定写 `localhost:5001`。

## 兼容性

- 服务器本机现有更新按钮和自动后台调度不受影响。
- 局域网其他电脑的图表、表格、搜索和数据读取不受影响。
- 远程页面上的写按钮会收到明确的 HTTP 403 和中文只读说明。
- 主要页面在收到 403 后立即恢复按钮并显示原因，不会误进入后台任务轮询或长期停留在“更新中”。
- 共享导航在只读模式下禁用更新、同步、回填、保存、补全和重新分类按钮；刷新、筛选和图表交互继续可用。
- 数据库、原始数据、缓存和已有 observation 均未修改。

## 验证

```powershell
python -m unittest tests.test_server_access_security
python -m unittest discover -s tests
```

重启服务后再进行浏览器验收，确认本机显示“本机管理模式”，校园网设备显示“局域网只读模式”。

本次实测结果：73 项单元测试全部通过；本机健康检查为 `manage`，校园网 IP 的 GET 返回 200、POST 返回 403；桌面和 390×844 移动视口均显示“局域网只读”，写操作按钮已禁用。控制台仅保留项目既有的浏览器内 Babel 开发模式警告。
