# 安居客挂牌房价侦察报告

> 侦察日期：2026-07-16（America/Denver）
> 全国入口：<https://www.anjuke.com/fangjia/>
> 城市页：`https://<city-subdomain>.anjuke.com/market/`

## 1. 结论

安居客数据在当前环境可以低频取得，但风控具有明显不稳定性：真实 Chrome 首次访问进入 58 验证页；改用一个持久 HTTP 会话、正常浏览器请求头、入口 Referer，并在城市请求间随机等待 2–4 秒后，全国入口和北京/上海/成都城市页均返回真实 HTML。

因此模块采用“低频持久会话 + 原始缓存 + blocked 即跳过”，不使用代理池、验证码识别、滑块破解或其他绕过措施。

## 2. 真实页面结构

### 2.1 全国城市映射

全国入口 HTML 约 133 KB。城市数据位于：

```text
window.__NUXT__.data[1].filterData.areaFilter
```

其中每个城市包含 `name` 与 `url_mark`。将其与本地主库 `housing_city_observations` 的 70 个 DISTINCT city 精确匹配，结果为 **70/70**；映射已固化在 `services/anjuke_city_map.py`。

### 2.2 城市挂牌字段

城市页为 Nuxt SSR，主要数据位于：

```text
window.__NUXT__.data[0].priceTrendReportData
  cityName
  priceInfo.month
  priceInfo.price
  priceInfo.monthChange
  priceInfo.yearChange
  priceTrend[trendType=3].city[]  # 月度 date + price，实测北京 36 个月
```

HTML 还直接渲染三个 `.price-item`：挂牌均价、比上月、比去年。解析器以 Nuxt 数据块为主，并用 2,000–200,000 元/㎡价格护栏拒绝错位值。

## 3. 三城冒烟结果

请求使用同一会话，城市间等待 2–4 秒，共 3 次页面请求，无验证码或重定向。

| 城市 | HTTP/页面 | 解析结果 | 处理 |
|---|---|---|---|
| 北京 | 200，真实 HTML | 当月字段和 36 个月序列均可解析；具体数值不写入仓库 | 成功写入独立本地库 |
| 上海 | 200，真实 HTML | `priceInfo={}`，页面三个值均为 `-` | `parse_fail`，不造数 |
| 成都 | 200，真实 HTML | `priceInfo={}`，页面三个值均为 `-` | `parse_fail`，不造数 |

北京同页历史值已与统计局同月官方二手同比对齐，并成功计算分歧；真实挂牌值及其衍生结果仅保留在本机独立库和本地 API，不写入仓库。

上海和成都的全国/年度索引页存在年度均价片段，但定义和时间粒度不同，不能混入当月挂牌表，因此保持缺失。

## 4. 反爬与失败策略

- 单个 `requests.Session` 复用 Cookie。
- 正常桌面浏览器 UA、Accept、Accept-Language、入口 Referer。
- 城市之间随机等待 2–4 秒。
- 网络错误或 5xx 最多退避重试 2 次；全轮总请求硬上限 100。
- 响应小于 5 KB，或包含“访问验证”“请输入验证码”“滑块验证”及 antibot URL 时，立即记录 `blocked`，该城市不重试。
- 正常 HTML 但数据字段为空单独记 `parse_fail`，不误报 blocked。
- 成功 HTML 缓存到 `data/anjuke_raw/<YYYY-MM>/`；同月缓存优先复用，减少重复请求。
- 原始 HTML 和 `data/housing_listing.db*` 均由 `.gitignore` 排除。
- 解析失败只追加日志，不删除旧 observation。
